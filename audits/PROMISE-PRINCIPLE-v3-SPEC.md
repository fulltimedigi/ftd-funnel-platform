# Promise Principle v3 — the Constraint Kernel (spec for ADR-0037)

Supersedes the ad-hoc hard-axis fixes (ADR-0034 form, ADR-0035 price): both become typed
constraints inside ONE kernel. Incorporates external design review. Scope = "correct core"
(closes every correctness hole; defers full live-matching re-architecture) plus two standing
architectural guardrails.

## Guardrails (from now on)
- **G1.** The deterministic **matcher (constraint kernel) is the single source of truth for
  what "matches" means.** The decision table is a *materialized artifact/cache generated by
  the kernel* — it carries no independent matching logic.
- **G2.** Every result carries `catalog_version` + `policy_version`, and a final deterministic
  re-verification runs on the chosen SKU **before display**.

## The kernel

**Three-valued status** — for each answer × recommendable unit:
`status(answer, unit) → SAT | VIOLATED | UNKNOWN`.
- Evaluate the axis's TYPED predicate against the unit's **grounded** value.
- Ungrounded / missing value → **UNKNOWN** (never silently treated as a match).

**Typed predicates** (answers compile to predicates, not label-equality):
- nominal: set membership (values are multi-valued sets; SAT if intersection ≠ ∅).
- taxonomy: requested is ancestor-or-equal of the unit's value (e.g. "spray perfume" ⊇ EDP/EDT).
- ordinal (e.g. roast light<medium<dark): SAT iff equal; else VIOLATED with magnitude = ordinal distance.
- numeric price: **directional** — for "≤ cap": SAT iff price ≤ cap; VIOLATED magnitude = max(0, price−cap); under-budget is cheaper than over-budget (distance model), never equal.
- variant/SKU: SAT only if ≥1 **purchasable variant** satisfies the variant-level answers (match the buyable SKU, not the product shell).

**Constraint modes & priority (separate fields):**
`ask_order` (UX flow) ≠ `relaxation_priority` (importance) ≠ `mode` ∈ {NEVER_RELAX, RELAXABLE, ADVISORY} ≠ `violation_distance`.
Mode/priority derive from answer semantics + wording, with smart defaults, merchant-overridable.
Defaults: product-form, strict budget cap, compatibility/allergen/safety → NEVER_RELAX.

**Selection = lexicographic partial-CSP (not "drop & retry" as the spec):**
1. Exclude any unit where a NEVER_RELAX constraint is VIOLATED **or UNKNOWN**.
2. For each survivor compute a `loss` vector:
   `< (conflicts@prio1, magnitude), (conflicts@prio2, magnitude), …, unknown_penalty, advisory_loss, stable_tiebreak >`.
3. Pick the lexicographically smallest loss. **Coverage is a tie-break ONLY, between equal-quality matches** — never overrides fit.
4. If an EXACT candidate exists, a COMPROMISE candidate may not be chosen (exact dominance).
5. Enforce relaxation **bounds** (max budget overshoot, no compatibility/safety/allergen relax, no UNKNOWN on an asserted match) → beyond a bound, return the **first-class "no exact match"** state, not a confident card with a footnote.

**Result states:** `EXACT` (all answers SAT) / `COMPROMISE` (≥1 VIOLATED) / `UNVERIFIED` (no VIOLATED but ≥1 relevant UNKNOWN).

**Disclosure** is computed by **re-comparing the final chosen unit to every answer** (fresh
status), NOT from the constraints the search happened to drop. List conflicts (VIOLATED) and
relevant unknowns, with distinct wording:
- conflict: "اخترت [medium]، وده [light]."
- unknown: "مقدرناش نتأكّد من [التحميص] للمنتج ده."

**Grounding is per unit × claim** (not per axis). Provenance order: structured field >
merchant-approved mapping > deterministic ontology from authoritative store text > validated
extraction (negation/qualifier-aware) > uncorroborated inference. Only the top tiers count as
"known" for a hard constraint; token-presence alone is NOT sufficient. One ungrounded product
does not demote a whole axis — that product is UNKNOWN on that axis.

## No bypass
NO global "best product" fallback, NO out-of-stock substitution, NO coverage/boost reranking
may produce a renderable result that skips the kernel. There is exactly ONE constructor of a
`SelectionResult`, and it requires a passing verification certificate.

## Result schema (carries the proof)
`SelectionResult { product_id, variant_id, catalog_version, policy_version, match_state,
matches[], conflicts[], unknowns[], tie_break_reason }`. Renderer consumes structured fields —
disclosure is never stored solely as authored prose (mobile/i18n/a11y cannot hide a compromise).

## Coverage (named honestly)
Guarantee **SURFACED coverage** (every SKU is primary OR a clearly labeled alternate). Alternates
also pass the kernel (respect NEVER_RELAX). Do not claim universal exact-primary coverage
(impossible with duplicate profiles / nulls).

## Honesty > depth
When the catalog can't honestly support depth, the funnel gets SHORTER — two honest questions
beat four with two lies. Redefine "rich catalog" as *enough trustworthy differentiating data*,
not product count. The Promise wins over the anti-bland/richness gates on thin catalogs.

## The guarantee (per-artifact, not just synthetic)
- **Publish-time exhaustive verification of every REAL funnel** (not only CI on synthetic
  stores): enumerate every reachable path of THAT merchant's artifact and assert the exit
  criteria below. A finite table → an exhaustive proof for that store.
- **Runtime cheap verifier** before render (re-check the SKU vs current catalog; on a
  NEVER_RELAX break → don't render, re-run kernel / mark stale).
- **Property-based / fuzz catalogs**: nulls, duplicate/dominated profiles, multi-label, numeric
  boundaries, single-product, all-identical, deleted-after-compile, price/stock mutation.
- **By-construction argument** for the rank-1 guarantee, written in the docs (2 lines).

### v3 EXIT CRITERIA — v3 is done only when, for every real path:
1. product & SKU exist in the served catalog version.
2. every NEVER_RELAX constraint = SAT.
3. every VIOLATED or UNKNOWN answer appears in the disclosure.
4. if an exact candidate exists, no compromise candidate is chosen.
5. the chosen unit is best per the declared violation vector.
6. no fallback / coverage reranking can corrupt that vector.

## Deferred (only until a real catalog proves the artifact unsafe)
Full runtime-live-matching rebuild and live stock/price dereferencing. (G1/G2 make the later
migration a change of *implementation*, not of *meaning*.)

## Gates to keep/add
Keep trust / anti-bland / richness. Add: exact-path-rate, max-relaxation-severity,
unknown-attribute-rate, prefix-support (each offered answer is viable given prior answers),
predicate-evidence-coverage, variant-validity, staleness/version, disclosure-rendering.

Prior-art framing to adopt (not a heavy solver dep): constraint-based recommendation
(Felfernig), constraint hierarchies (Borning), lexicographic/partial CSP, QuickXplain / minimal
correction sets for relaxation+explanation, dynamic/conditional CSP, faceted navigation (every
filter hard by default — our bug was deviating from that default).
